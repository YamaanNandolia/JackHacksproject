# JacCoder Companion — VS Code Extension

Detects JacCoder job workspaces, surfaces build context, discovers the
JacCoder extension, and attempts to open a new session automatically.
Falls back to an AppleScript-driven keystroke sequence on macOS if direct
command execution is unavailable.

---

## What it does

| Behaviour | Detail |
|-----------|--------|
| **Activates** | On `onStartupFinished` — zero cost when no job workspace is open |
| **Detects** | Scans all open workspace folders for `.jaccoder-job.json` |
| **Logs job info** | ID, app name, slug, language, entry point, features, launch time, prompt preview |
| **Discovers JacCoder** | Scans all installed extensions; scores by ID/display-name keywords (`jaccoder`, `claude`, `anthropic`, …) |
| **Logs extension metadata** | ID, version, publisher, install path, every contributed command ID |
| **Direct trigger** | Attempts likely "open/new session" commands in priority order; passes the job prompt as an argument |
| **AppleScript fallback** | If direct trigger fails (macOS only): focuses VS Code, opens Command Palette, runs the JacCoder command, pastes `PROMPT.md`, presses Return |
| **Output channel** | All activity timestamped in **Output › JacCoder** |

## What it does NOT do

- Run `jac` or build the application
- Provide tree views, code lens, or editor commands
- Make network requests
- Support non-macOS AppleScript fallback (skipped gracefully on Linux/Windows)

## One-time setup

### 1. Install and compile

```bash
cd automation/companion_extension
npm install
npm run compile        # outputs to out/
```

Watch mode for active development:
```bash
npm run watch
```

### 2. Sideload into VS Code

```bash
# From automation/companion_extension/
ln -s "$(pwd)" ~/.vscode/extensions/jaccoder-companion
```

Reload VS Code: **⌘⇧P** → `Developer: Reload Window`

### 3. Verify it loaded

**View › Extensions** → search `JacCoder Companion` → must show as **Enabled**

---

## macOS permissions (for AppleScript fallback)

The fallback uses `System Events` to send keystrokes to VS Code.

Grant once in **System Preferences › Privacy & Security**:

| Section | What to allow |
|---------|---------------|
| **Accessibility** | Visual Studio Code |
| **Automation** | Terminal (or whichever app calls `osascript`) → System Events |
| **Automation** | Terminal → Visual Studio Code |

macOS will show a permission prompt on first use. If you dismissed it, open
System Preferences and add the entries manually.

## Verifying it works

1. Run a job from the project root:
   ```bash
   ./automation/launcher/run_job.sh jobs/sample_job.json
   ```
2. VS Code opens the generated workspace automatically.
3. Open **View › Output** → select **JacCoder**.
4. You should see:
   - Job metadata block
   - Extension discovery results (list of scanned extensions + scores)
   - Metadata for the discovered JacCoder extension and all its command IDs
   - Each command attempt and its result
   - AppleScript invocation (if triggered)

## Packaging as a `.vsix` (optional)

```bash
npm install -g @vscode/vsce
vsce package
code --install-extension jaccoder-companion-0.1.0.vsix
```

> **Note:** When installed from a `.vsix`, the AppleScript fallback resolves
> the script path relative to `context.extensionPath`. If you move or
> repackage the extension, ensure `focus_and_paste.applescript` is accessible
> at `<extensionPath>/../launcher/focus_and_paste.applescript`.
